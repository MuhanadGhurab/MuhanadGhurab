# Homepage Architecture

## Visitor journey (5–10 seconds)

1. See name + cyber crow hero + domain triad  
2. Confirm role (IT Specialist — Tekfen) and location  
3. Scan featured work / Start Here paths  

## Section purpose

| Section | Purpose |
|---------|---------|
| Cyber Crow Hero | Identity and brand lock |
| Snapshot | Recruiter scan of facts |
| Terminal | Compact technical self-intro |
| Overview | Narrative context (~100 words) |
| Featured work | Strongest evidence first |
| Evidence strip | Proof without fake totals |
| Capability matrix | Categories without skill % |
| Operating status | Project/learning completion only |
| Learning | Security+ / PMP honest status |
| Achievements | Selective, non-inflated |
| Start Here | Role-based navigation |
| Contact | CTA |

## Recruiter journey

Snapshot → Featured (lab / SecureSkies / tools) → Start Here matching requisition → Contact

## Technical reviewer journey

Featured deep links → Evidence strip → Capability matrix → Repository docs and CI

## Project selection logic

Feature only verified public repositories with documentation or tests. Maximum six cards. Prefer cybersecurity evidence over inventory completeness. Planning stubs stay out of the primary surface.

## Information hierarchy

Identity > Role > Evidence > Skills categories > Progress > Learning > Achievements > Navigation > Contact
